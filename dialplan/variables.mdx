## What are Variables?

Variables in Asterisk store data that can be used throughout the dialplan. They enable dynamic call routing, store user input, retrieve database values, and pass information between contexts and extensions.

<Info>
Variables make your dialplan dynamic and powerful. Instead of hardcoding values, you can make decisions based on caller information, user input, database lookups, and more.
</Info>

## Variable Syntax

Reference variables using the `${}` syntax:

```conf
${VARIABLE_NAME}
```

Example:

```conf extensions.conf
exten => 4000,1,NoOp(Calling ${EXTEN})
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
```

## Built-in Channel Variables

Asterisk provides many built-in variables automatically:

### ${EXTEN} - Dialed Extension

The extension that was dialed:

```conf extensions.conf
[empleado]
exten => 4000,1,NoOp(Llamando a Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})  ; Uses 4000 as mailbox
same => n,Hangup()

exten => _*XXX,1,Set(llamante=${CALLERID(num)})
same => n,Goto(buzon-voz,1,1)  ; ${EXTEN} would be like *4000
```

#### Extracting Parts of ${EXTEN}

Use substring notation `${VARIABLE:offset:length}`:

```conf extensions.conf
; Extract last 3 digits from 9-digit number
exten => _995554XXX,1,Set(extension=${EXTEN:5})  ; Skip first 5 digits
same => n,NoOp(Calling extension ${extension})
same => n,Goto(llamar,${extension},1)

; Skip first 2 characters (*8 prefix)
exten => _*8.,1,Set(extension=${EXTEN:2})
same => n,NoOp(Capturando llamada a la extension ${extension})
same => n,Pickup(${extension})

; Get first character
exten => _*8.,1,Set(extension=${EXTEN:2})
same => n,Set(pref=${extension:0:1})  ; First digit
same => n,GotoIf($["${pref}" = "3"]?llamar,8,1)
```

<Tip>
`${EXTEN:2}` means "skip the first 2 characters". `${EXTEN:0:1}` means "get 1 character starting at position 0".
</Tip>

### ${CALLERID()} - Caller ID Information

Get information about the caller:

```conf extensions.conf
; Get caller number
exten => _*XXX,1,Set(llamante=${CALLERID(num)})
same => n,Goto(buzon-voz,1,1)

; Get caller name and number
exten => 101,1,NoOp(CALLERID = ${CALLERID(name)}-------${CALLERID(number)})
same => n,Hangup()

; Get ANI (Automatic Number Identification)
exten => _X.,1,NoOp(ANI: ${CALLERID(ani)})
same => n,SayDigits(${CALLERID(ani)})
```

**Common CALLERID() fields:**
- `${CALLERID(num)}` - Caller's phone number
- `${CALLERID(name)}` - Caller's name
- `${CALLERID(ani)}` - ANI (calling number)
- `${CALLERID(all)}` - Full caller ID string

### ${MESSAGE()} - SIP Message Data

For SIP MESSAGE handling:

```conf extensions.conf
[mensajes]
exten => _X.,1,NoOp(Mensaje de ${MESSAGE(from)})
same => n,NoOp(Mensaje para ${MESSAGE(to)})
same => n,NoOp(Cuerpo del mensaje: ${MESSAGE(body)})
same => n,Set(dest=${EXTEN})
same => n,Set(remitente=${CUT(MESSAGE(from),<,2)})
same => n,Set(texto=${MESSAGE(body)})
same => n,Set(MESSAGE(body)=${remitente}: ${texto})
same => n,MessageSend(sip:${dest}, Centro de Mensajes(CdM))
```

**MESSAGE() fields:**
- `${MESSAGE(from)}` - Sender URI
- `${MESSAGE(to)}` - Recipient URI
- `${MESSAGE(body)}` - Message text (can be set)

### ${MESSAGE_SEND_STATUS} - Message Status

Status after MessageSend():

```conf extensions.conf
[mensajes]
exten => _X.,1,MessageSend(sip:${dest}, Centro de Mensajes(CdM))
same => n,Noop(Estado del mensaje ${MESSAGE_SEND_STATUS})
same => n,GotoIf($["${MESSAGE_SEND_STATUS}" != "SUCCESS"]?fallo,s,1)
same => n,Hangup()
```

### ${DIALSTATUS} - Dial Result

Result of Dial() application:

```conf extensions.conf
exten => s,1,Dial(${ARG1})
same => n,Goto(s-${DIALSTATUS},1)

exten => s-NOANSWER,1,Hangup()
exten => s-BUSY,1,Hangup()

exten => stdexten-NOANSWER,1,VoiceMail(${mbx},u)
exten => stdexten-BUSY,1,VoiceMail(${mbx},b)
```

**Common DIALSTATUS values:**
- `ANSWER` - Call was answered
- `BUSY` - Line was busy
- `NOANSWER` - No answer
- `CANCEL` - Caller hung up
- `CONGESTION` - Network congestion
- `CHANUNAVAIL` - Channel unavailable

### ${CHANNEL()} - Channel Information

Set or get channel-specific information:

```conf extensions.conf
[recepcion]
; Set music on hold class
exten => 1,1,Set(CHANNEL(musicclass)=recepcion)
same => n,Answer()
same => n,Queue(registrados,,,,60)

[demo]
; Set language
exten => 3,1,Set(CHANNEL(language)=fr)
same => n,Goto(s,restart)
```

**Common CHANNEL() fields:**
- `${CHANNEL(musicclass)}` - Music on hold class
- `${CHANNEL(language)}` - Channel language
- `${CHANNEL(accountcode)}` - Account code for billing

### Other Built-in Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `${EPOCH}` | Current Unix timestamp | `1616161616` |
| `${UNIQUEID}` | Unique call identifier | `1616161616.123` |
| `${CONTEXT}` | Current context | `empleado` |
| `${PRIORITY}` | Current priority | `3` |
| `${EXTEN}` | Current extension | `4000` |

## Setting Variables with Set()

The `Set()` application creates or modifies variables.

**Syntax:** `Set(variable=value)`

### Basic Variable Assignment

```conf extensions.conf
; Simple assignment
exten => 1,1,Set(nombre=Javier)
same => n,NoOp(Hola ${nombre})

; Assign from another variable
exten => _XXX,1,Set(extension=${EXTEN})
same => n,Set(nombre=${ODBC_extension(${extension})})

; Assign from pattern extraction
exten => _*8.,1,Set(extension=${EXTEN:2})
same => n,Set(pref=${extension:0:1})
```

### Examples from Tutorial

#### Store Caller Number

```conf extensions.conf
[recepcion]
exten => _*XXX,1,Set(llamante=${CALLERID(num)})
same => n,Goto(buzon-voz,1,1)
same => n,Hangup()
```

#### Extract Extension from Long Number

```conf extensions.conf
[entrantes]
exten => _995554XXX,1,Set(extension=${EXTEN:5})
same => n,NoOp(Llamando a la extension ${extension})
same => n,Goto(llamar,${extension},1)
```

#### Message Processing

```conf extensions.conf
[mensajes]
exten => _X.,1,Set(dest=${EXTEN})
same => n,Set(remitente=${CUT(MESSAGE(from),<,2)})
same => n,Set(remitente=${CUT(remitente,@,1)})
same => n,Set(remitente=${CUT(remitente,:,2)})
same => n,Set(texto=${MESSAGE(body)})
same => n,Set(MESSAGE(body)=${remitente}: ${texto})
```

## String Manipulation Functions

Asterisk provides functions for manipulating variables:

### CUT() - Extract Substring

Extract part of a string by delimiter.

**Syntax:** `${CUT(variable,delimiter,field)}`

```conf extensions.conf
[mensajes]
; Extract domain from SIP URI: sip:user@domain.com
exten => _X.,1,Set(remitente=${CUT(MESSAGE(from),<,2)})  ; Get part after <
same => n,Set(remitente=${CUT(remitente,@,1)})            ; Get part before @
same => n,Set(remitente=${CUT(remitente,:,2)})            ; Get part after :

[fallo]
exten => s,1,Set(remit=${CUT(MESSAGE(from),<,2)})
same => n,Set(remit=${CUT(remit,@,1)})
```

<Info>
CUT() is 1-indexed. Field 1 is the first segment, field 2 is the second, etc. Use `-1` for the last field.
</Info>

### FILTER() - Filter Characters

Keep only specified characters.

**Syntax:** `${FILTER(allowed_chars,string)}`

```conf extensions.conf
; Keep only digits
exten => _9011.,1,Dial(${GLOBAL(TRUNK)}/${FILTER(0-9,${EXTEN:${GLOBAL(TRUNKMSD)}})})
```

### IF() - Conditional Expression

Return one value or another based on condition.

**Syntax:** `${IF(condition?true_value:false_value)}`

```conf extensions.conf
[mensajes]
exten => _X.,1,Set(nombre=${ODBC_extension(${remitente})})
same => n,Set(nombre=${IF($["${nombre}" = ""]?desconocido)})
```

## Global Variables

Global variables are accessible from all contexts. Define them in the `[globals]` section:

```conf extensions.conf
[globals]
CONSOLE=Console/dsp
IAXINFO=guest
TRUNK=DAHDI/G2
TRUNKMSD=1
```

Reference with `${GLOBAL()}`:

```conf
exten => 1234,1,Dial(${GLOBAL(CONSOLE)})
exten => _9011.,1,Dial(${GLOBAL(TRUNK)}/${EXTEN:${GLOBAL(TRUNKMSD)}})
```

## Database Integration with ODBC

Asterisk can query databases and store results in variables:

### ODBC_* Functions

The tutorial uses ODBC functions to query MariaDB:

```conf extensions.conf
[abierto]
; Look up user by ID
exten => 10,1(id),Festival(Por favor marque su numero de identificacion)
same => n,Read(id,,4,3,5)
same => n,Set(nombre=${ODBC_usuario(${id})})
same => n,GotoIf($["${nombre}" != ""]?recepcion,1,1)
same => n,Festival(Usario no encontrado)

[recepcion]
; Look up extension name
exten => _XXX,1,Set(extension=${EXTEN})
same => n,Set(nombre=${ODBC_extension(${extension})})
same => n,GotoIf($["${nombre}" != ""]?llamar,${extension},1)

[empleado]
; Look up info on caller
exten => 100,1,NoOp(Esta llamando ${ODBC_info(${CALLERID(number)})})
same => n,Festival(Va a escuchar un fragmento de la primavera de Antonio Vivaldi)

[buzon-voz]
; Verify PIN
exten => 1,1(pin),Festival(Marque su numero de identificacion privado)
same => n,Read(pin,,4,3,5)
same => n,Set(idpin=${ODBC_pin(${llamante})})
same => n,GotoIf($["${pin}" = "${idpin}"]?2,1)

[llamar]
; Get user's team/context
exten => 8,1,Festival(Capturando llamada a la extension ${extension})
same => n,Set(equipo=${ODBC_equipo(${extension})})
same => n,Pickup(${extension}@${equipo})
```

<Info>
ODBC functions are defined in `/etc/asterisk/func_odbc.conf` and map to SQL queries. The function name becomes available as `${ODBC_functionname(arg)}`.
</Info>

### Example func_odbc.conf

```conf func_odbc.conf
[usuario]
dsn=MySQL-asterisk
readsql=SELECT nombre FROM usuarios WHERE id='${ARG1}'

[extension]
dsn=MySQL-asterisk
readsql=SELECT nombre FROM extensiones WHERE numero='${ARG1}'

[pin]
dsn=MySQL-asterisk
readsql=SELECT pin FROM usuarios WHERE extension='${ARG1}'

[equipo]
dsn=MySQL-asterisk
readsql=SELECT equipo FROM extensiones WHERE numero='${ARG1}'
```

## Variable Scope

### Channel Variables

Standard variables - exist only for current channel:

```conf
Set(extension=4000)  ; Only this channel
```

### Global Variables

Shared across all channels:

```conf
Set(GLOBAL(myvar)=value)  ; All channels can access
```

### Inherited Variables

Passed to child channels (prefix with `__`):

```conf
Set(__SIPFROMUSER=${CALLERID(num)})    ; Inherited by bridged channel
Set(__SIPFROMDOMAIN=${GLOBAL(FREENUMDOMAIN)})
```

## Complex Variable Examples

### Multi-Step Variable Processing

```conf extensions.conf
[mensajes]
; Parse SIP MESSAGE sender
exten => _X.,1,Set(dest=${EXTEN})
same => n,Set(remitente=${CUT(MESSAGE(from),<,2)})  ; Get URI part
same => n,Set(remitente=${CUT(remitente,@,1)})      ; Remove domain
same => n,Set(remitente=${CUT(remitente,:,2)})      ; Remove 'sip:'
same => n,Set(texto=${MESSAGE(body)})               ; Get message text
same => n,Set(MESSAGE(body)=${remitente}: ${texto}) ; Prepend sender
```

### Time-Based Variables

```conf extensions.conf
[time]
exten => _X.,1,NoOp(Time: ${EXTEN} ${timezone})
same => n,Set(FUTURETIME=$[${EPOCH} + 12])
same => n,SayUnixTime(${FUTURETIME},Zulu,HNS)
```

### Loop Control

```conf extensions.conf
[abierto]
exten => 1,1,Answer()
same => n(loop),Festival(Presione 1 o 2)
same => n,Read(opcion,,1,3)
same => n,SayNumber(${opcion})
same => n,GotoIf($["${opcion}" = "1"]?10,1)
same => n,GotoIf($["${opcion}" = "2"]?20,1)
same => n,Festival(Opcion incorrecta)
same => n,Goto(1,loop)  ; Loop back using label
```

## Viewing Variables

From the Asterisk CLI during an active call:

```
CLI> core show channels
CLI> core show channel <channel_name>
CLI> core show channelvars <channel_name>
```

From dialplan with NoOp():

```conf
exten => 100,1,NoOp(EXTEN=${EXTEN} CALLERID=${CALLERID(num)})
exten => 101,1,NoOp(extension=${extension} nombre=${nombre})
```

## Best Practices

<Steps>
  <Step title="Use descriptive names">
    Name variables clearly: `extension`, `nombre`, `pin` rather than `x`, `n`, `p`
  </Step>
  
  <Step title="Check for empty values">
    Always validate database lookups and user input
    
    ```conf
    GotoIf($["${nombre}" != ""]?valid)
    ```
  </Step>
  
  <Step title="Log variable values">
    Use NoOp() to debug variable contents
    
    ```conf
    NoOp(extension=${extension} nombre=${nombre})
    ```
  </Step>
  
  <Step title="Use globals sparingly">
    Limit global variables to truly global values (trunks, system settings)
  </Step>
  
  <Step title="Document ODBC functions">
    Comment what each database function returns
  </Step>
</Steps>

<Tip>
When debugging, add NoOp() statements to log variable values at each step. This makes it much easier to trace execution flow and find issues.
</Tip>
