## Extension Basics

An **extension** is a sequence of actions that Asterisk executes when a specific number or pattern is dialed. Extensions are the building blocks of the dialplan.

### Basic Syntax

```conf
exten => extension_name,priority,Application(arguments)
```

Example from the tutorial:

```conf extensions.conf
exten => 4000,1,NoOp(Llamando a Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()
```

## Extension Components

<Steps>
  <Step title="Extension Pattern">
    The number or pattern that triggers this extension (e.g., `4000`, `_XXX`, `*98`)
  </Step>
  
  <Step title="Priority">
    The execution order: numeric (1, 2, 3), `n` (next), or `same` (same extension)
  </Step>
  
  <Step title="Application">
    The action to perform (Dial, VoiceMail, Playback, etc.)
  </Step>
  
  <Step title="Arguments">
    Parameters passed to the application
  </Step>
</Steps>

## Priority Types

Asterisk supports three ways to specify priorities:

<CodeGroup>
```conf Explicit Numeric
[empleado]
exten => 4000,1,NoOp(First step)
exten => 4000,2,Dial(SIP/javier,15)
exten => 4000,3,VoiceMail(${EXTEN})
exten => 4000,4,Hangup()
```

```conf 'n' for Next
[empleado]
exten => 4000,1,NoOp(First step)
exten => 4000,n,Dial(SIP/javier,15)
exten => 4000,n,VoiceMail(${EXTEN})
exten => 4000,n,Hangup()
```

```conf 'same' for Same Extension (Modern)
[empleado]
exten => 4000,1,NoOp(First step)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()
```
</CodeGroup>

<Tip>
Using `same => n` is the modern, recommended syntax. It's cleaner and reduces errors when renumbering extensions.
</Tip>

## Pattern Matching

Pattern matching allows one extension to match multiple dialed numbers. Patterns always start with an underscore `_`.

### Pattern Matching Characters

| Pattern | Matches | Example |
|---------|---------|----------|
| `X` | Any digit 0-9 | `_XXX` matches 100-999 |
| `Z` | Any digit 1-9 | `_ZXX` matches 100-999 (not 000-099) |
| `N` | Any digit 2-9 | `_NXX` matches 200-999 |
| `.` | One or more of previous | `_X.` matches 1, 12, 123, 1234... |
| `!` | Zero or more of anything | `_X!` matches 1, 12, abc, 1a2b... |
| `[]` | Character range | `_[1-3]XX` matches 100-399 |

### Pattern Examples from Tutorial

#### Three-Digit Extensions

Match any 3-digit extension:

```conf extensions.conf
[recepcion]
; Call any 3-digit extension
exten => _XXX,1,Set(extension=${EXTEN})
same => n,Set(nombre=${ODBC_extension(${extension})})
same => n,GotoIf($["${nombre}" != ""]?llamar,${extension},1)
same => n,Festival(La extension ${extension} no se encuentra disponible)
same => n,Hangup()
```

<Info>
`_XXX` matches extensions 000 through 999. The dialed number is stored in `${EXTEN}`.
</Info>

#### Voicemail Access

Match extensions starting with asterisk:

```conf extensions.conf
[recepcion]
; Access voicemail for any extension
exten => _*XXX,1,Set(llamante=${CALLERID(num)})
same => n,Goto(buzon-voz,1,1)
same => n,Hangup()
```

Dialing `*123` captures the extension and routes to voicemail context.

#### Call Pickup Pattern

Call pickup with *8 prefix:

```conf extensions.conf
[empleado]
; Capture call to any extension with *8
exten => _*8.,1,NoOp(Capturando llamada a la extension ${EXTEN:2})
same => n,Pickup(${EXTEN:2})
same => n,Hangup()
```

Dialing `*84000` picks up the call ringing at extension 4000.

#### Group Calling Pattern

Match group extensions (X00 pattern):

```conf extensions.conf
[direccion]
; Call groups: 100, 200, 300, 400, etc.
exten => _X00,1,Festival(Llamada a grupo ${EXTEN})
same => n,Goto(llamar-grupo,${EXTEN},1)
same => n,Hangup()

[llamar-grupo]
; Reception group
exten => 100,1,Dial(SIP/101&SIP/102&SIP/103&SIP/110,60)
same => n,Hangup()

; Management group
exten => 200,1,Dial(SIP/201&SIP/202&SIP/203,60)
same => n,Hangup()

; Doctors group
exten => 300,1,Dial(SIP/301&SIP/302&SIP/303,60)
same => n,Hangup()
```

#### External Number Pattern

Match 9-digit phone numbers (Spanish format):

```conf extensions.conf
[direccion]
; External calls (9 digits)
exten => _XXXXXXXXX,1,Goto(llamar,${EXTEN},1)
same => n,Hangup()
```

#### Variable Length Matching

Catch-all pattern for SIP messages:

```conf extensions.conf
[mensajes]
; Match any numeric extension
exten => _X.,1,Set(dest=${EXTEN})
same => n,Set(remitente=${CUT(MESSAGE(from),<,2)})
same => n,Set(remitente=${CUT(remitente,@,1)})
same => n,Set(remitente=${CUT(remitente,:,2)})
same => n,Set(texto=${MESSAGE(body)})
same => n,Set(MESSAGE(body)=${remitente}: ${texto})
same => n,MessageSend(sip:${dest}, Centro de Mensajes(CdM))
same => n,Hangup()
```

<Tip>
`_X.` matches any string starting with a digit, followed by zero or more characters: 1, 12, 123, 1234, etc.
</Tip>

## Pattern Priority

When multiple patterns could match, Asterisk uses most-specific-first ordering:

```conf extensions.conf
[test]
exten => 123,1,NoOp(Exact match)              ; Priority 1: Exact
exten => _12X,1,NoOp(Three digit match)        ; Priority 2: Specific pattern
exten => _1XX,1,NoOp(Hundreds match)           ; Priority 3: Less specific
exten => _XXX,1,NoOp(Any three digits)         ; Priority 4: Generic
exten => _X.,1,NoOp(Any number)                ; Priority 5: Catch-all
```

Dialing `123` matches the exact extension first, not the patterns.

## Special Extensions

Asterisk provides special extension names:

### s - Start Extension

The default entry point when entering a context:

```conf extensions.conf
[demo]
exten => s,1,Wait(1)
same => n,Answer()
same => n,Set(TIMEOUT(digit)=5)
same => n,Set(TIMEOUT(response)=10)
same => n(restart),BackGround(demo-congrats)
same => n,WaitExten()
```

### i - Invalid Extension

Handles invalid dialed numbers:

```conf extensions.conf
[empleado]
exten => i,1,Playback(invalid)
same => n,Hangup()
```

### t - Timeout Extension

Handles when caller doesn't dial anything:

```conf extensions.conf
[demo]
exten => t,1,Goto(#,1)  ; If they take too long, give up
```

### h - Hangup Extension

Executes when call is disconnected:

```conf extensions.conf
[empleado]
exten => h,1,NoOp(Call ended at ${EPOCH})
same => n,System(/usr/local/bin/log-call.sh ${UNIQUEID})
```

## Named Priorities

You can label priorities for use with Goto():

```conf extensions.conf
[abierto]
exten => 1,1,Answer()
same => n,Festival(Bienvenido al centro de salud)
same => n(loop),Festival(Presione)
same => n,SayNumber(1)
same => n,Festival(si es un usuario registrado o)
same => n,SayNumber(2)
same => n,Festival(en caso contrario)
same => n,Read(opcion,,1,3)
same => n,GotoIf($["${opcion}" = "1"]?10,1)
same => n,GotoIf($["${opcion}" = "2"]?20,1)
same => n,Festival(Opcion incorrecta)
same => n,Goto(1,loop)  ; Jump back to the (loop) label
same => n,Hangup()
```

<Info>
The label `(loop)` allows jumping back to that specific priority without knowing its numeric position.
</Info>

## Extension Examples from Tutorial

### Simple Extension

Basic call routing with voicemail:

```conf extensions.conf
[empleado]
exten => 4000,1,NoOp(Llamando a Javier)
same => n,Dial(SIP/javier,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()

exten => 4001,1,NoOp(Llamando a Belen)
same => n,Dial(SIP/belen,15)
same => n,VoiceMail(${EXTEN})
same => n,Hangup()
```

### Voicemail Access Extensions

Direct voicemail access:

```conf extensions.conf
[empleado]
exten => *4000,1,VoiceMailMain(4000)
same => n,Hangup()

exten => *4001,1,VoiceMailMain(4001)
same => n,Hangup()
```

### Music Playback Extension

Simple IVR with music:

```conf extensions.conf
[empleado]
exten => 100,1,NoOp(Centralita de distribucion de llamadas)
same => n,Playback(silence/1)
same => n,Playback(mimusica/primavera)
same => n,Hangup()
```

### Group Call Extension

Ring multiple phones simultaneously:

```conf extensions.conf
[entrantes]
exten => 4040,1,NoOp(Llamando al grupo 'empleados')
same => n,Dial(SIP/javier&SIP/belen,60)
same => n,Hangup()
```

### Time-Based Routing

Route calls based on business hours:

```conf extensions.conf
[exterior]
exten => 757112233,1,GotoIfTime(8:00-20:00,mon-fri,*,*?abierto,1,1)
same => n,Festival(En estos momentos no hay nadie que te pueda atender)
same => n,Festival(Nuestro horario es de 8 a 20 horas de lunes a viernes)
same => n,Hangup()
```

### Interactive Menu

IVR with user input:

```conf extensions.conf
[abierto]
exten => 1,1,Answer()
same => n,Festival(Bienvenido al centro de salud)
same => n(loop),Festival(Presione)
same => n,SayNumber(1)
same => n,Festival(si es un usuario registrado o)
same => n,SayNumber(2)
same => n,Festival(en caso contrario)
same => n,Read(opcion,,1,3)
same => n,SayNumber(${opcion})
same => n,GotoIf($["${opcion}" = "1"]?10,1)
same => n,GotoIf($["${opcion}" = "2"]?20,1)
same => n,Festival(Opcion incorrecta)
same => n,Goto(1,loop)
same => n,Hangup()
```

## Extension Testing

Test extension matching from the CLI:

```bash
asterisk -rx "dialplan show empleado"
asterisk -rx "dialplan show 4000@empleado"
```

Test pattern matching:

```
CLI> dialplan show _XXX@recepcion
CLI> dialplan show _*8.@empleado
```

## Best Practices

<Steps>
  <Step title="Use same => n syntax">
    Modern, cleaner, and easier to maintain than explicit priorities
  </Step>
  
  <Step title="Order patterns by specificity">
    Place exact matches before patterns, specific patterns before generic ones
  </Step>
  
  <Step title="Always end with Hangup()">
    Explicitly terminate calls to avoid undefined behavior
  </Step>
  
  <Step title="Use NoOp() for debugging">
    Add NoOp() statements to log execution flow
  </Step>
  
  <Step title="Use named labels">
    Label important priorities for easier Goto() navigation
  </Step>
</Steps>

<Tip>
Test your patterns thoroughly! Use `dialplan show` to verify which extension will match a given number.
</Tip>
